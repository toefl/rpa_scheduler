## 1. `get_next_task_to_run.pix`

Скрипт определяет, нужно ли запускать проекты на основе расписания (cron‑выражений), приоритета и флага `enabled`, формирует список активных проектов и вычисляет для каждого момент следующего триггера. В выбранный для запуска проект вносит обновление времени последнего запуска, а в `global_context` добавляет сведения о выбранном проекте и его стартовом скрипте.

| Направление | Аргумент | Тип | Описание |
| :--- | :--- | :--- | :--- |
| **InOut** | `global_context` | Dictionary<string, object> | Глобальный контекст, должен содержать ключ `schedule` (список проектов), значение по умолчанию: `new Dictionary<string, object>()`. |
| **Out** | `global_context["selected_project_name"]` | string | Имя выбранного для запуска проекта. |
| **Out** | `global_context["main_file"]` | string | Путь к стартовому скрипту выбранного проекта. |
| **InOut** | `global_context["schedule"]` | JsonElement | Глобальное расписание проектов; для выбранного проекта поле `last_run` обновляется текущим временем (в строковом представлении). |

1. Проверяет необходимость запуска проектов по расписанию, приоритету и флагу `enabled`.
2. Формирует список активных проектов и вычисляет для каждого момент следующего триггера по cron‑выражению.
3. Выбирает подходящий проект для запуска:  
   3.1. записывает его имя в `global_context["selected_project_name"]`;  
   3.2. записывает путь к его стартовому скрипту в `global_context["main_file"]`;  
   3.3. обновляет в `global_context["schedule"]` поле `last_run` выбранного проекта.
4. Если подходящих задач нет, только логирует информацию, не изменяя ключи выбора проекта.
5. Сохраняет обновлённое расписание в файл `schedule.json` как побочный эффект.

> **Условие:** В `global_context` должен быть корректно задан ключ `schedule`; при отсутствии подходящих задач изменения ограничиваются логированием, выбор проекта не выполняется.

## 2. `main.pix`

`RPA Scheduler` — централизованный планировщик задач для PIX‑роботов, который используется как единая точка входа вместо множества задач в Планировщике Windows. Планировщик автоматически определяет, какую задачу запустить следующей, и по каждому запуску последовательно выполняет ряд шагов.

| Направление | Аргумент | Тип | Описание |
| :--- | :--- | :--- | :--- |
|  |  | | Скрипт не возвращает данных; используется только как управляющая точка входа для подчинённых задач. |

1. Определяет следующую задачу к запуску на основе:
   1.1. текущего времени;  
   1.2. расписания (cron‑выражений);  
   1.3. приоритетов;  
   1.4. времени последнего запуска.
2. При срабатывании триггера (например, из Windows Task Scheduler) выполняет:  
   2.1. анализ конфигурации всех подчинённых задач;  
   2.2. выбор задачи, которая должна выполняться в данный момент;  
   2.3. запуск соответствующего скрипта `main.pix` и ожидание окончания его работы;  
   2.4. обновление поля `last_run` выбранной задачи и завершение собственной работы.

> **Условие:** Скрипт не принимает входные данные и не возвращает бизнес‑результатов, а только управляет запуском подчинённых задач по расписанию и приоритетам.
